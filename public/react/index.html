创建 React 应用

创建了一个名为 moz-todo-react 的文件夹作为你的项目文件夹

1

npx create-react-app moz-todo-react

React 在开发模式下会使用 严格模式（React.StrictMode），这会导致组件在 首次渲染 时被调用两次。这是为了帮助开发者发现潜在的副作用或问题，比如不安全的生命周期方法或不期望的副作用。

在你的代码中，App 组件中使用了 console.log(props) 来打印 props，由于 React.StrictMode 的启用，React 会执行双重调用，以确保组件在没有副作用的情况下能够正确运行。具体的行为如下：

首次渲染（Mount） 时，React 会调用 App 组件，并传入 props（即 {subject: ‘Clarice’}），并将它渲染到页面。 严格模式 触发的第二次渲染：在开发模式下，React 会再次调用 App 组件，确保在双重渲染下不会出现副作用问题。 因此，console.log(props) 会打印两次 {subject: ‘Clarice’}。

解决方法 如果你只是调试阶段，直接忽略这个行为即可。它只会在开发模式中出现，生产模式下不会进行双重渲染。

如何关闭严格模式： 如果你想暂时避免这个行为，可以将 React.StrictMode 删除或注释掉：

jsx Copy code root.render( // <React.StrictMode> // </React.StrictMode> ); 这样就不会触发双重渲染了，但建议在开发阶段保留严格模式以帮助你检测潜在的问题。 在开发模式下，React 的 严格模式（React.StrictMode）会使组件渲染两次，这是为了帮助开发者发现潜在的副作用和不安全的生命周期方法。虽然这个行为可能在调试阶段有些困惑，但它有几个重要的目的：

  1. 检测副作用 React 的严格模式会使组件在第一次渲染时，重新渲染一次。这可以帮助检测副作用问题，例如：

副作用：如果组件的副作用（例如网络请求、事件监听）在首次渲染时改变了组件状态，那么它可能会在第二次渲染中产生不可预测的结果。严格模式会暴露这种问题。 2. 验证组件 通过双重渲染，React 可以确保组件的 render 方法是 纯净的（即它不会改变组件的状态或进行其他副作用）。这有助于发现以下问题：

不安全的生命周期方法：比如在旧的生命周期方法中有副作用，可能导致不可预料的行为。 错误的副作用处理：在渲染过程中修改状态或进行其他操作，可能会导致不可预测的结果。 3. 检测不安全的生命周期方法 某些生命周期方法（如 componentWillMount、componentWillUpdate）在 React 的未来版本中被弃用或不推荐使用。严格模式会触发对这些不安全生命周期方法的调用，帮助开发者识别并修复这些问题。

  1. 帮助开发者找到潜在问题 通过在开发环境中暴露潜在问题，React 能够帮助开发者在生产环境之前解决这些问题，从而提高应用的稳定性和性能。

  2. 确认 React 18 的并发功能 React 18 引入了并发功能，这可能会影响组件的渲染行为。严格模式的双重渲染可以帮助开发者了解并发渲染下的组件行为，以便更好地适应这些新特性。

总结 双重渲染的目的是为了在开发阶段捕获潜在的问题，使得你可以在应用进入生产环境之前解决这些问题。这个行为不会出现在生产模式下，因此不影响最终用户的体验。
